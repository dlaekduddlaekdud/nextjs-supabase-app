# 모임 이벤트 관리 MVP PRD

> 작성일: 2026-02-25 | 버전: 0.1 (Draft) | 레포: nextjs-supabase-app/

---

## 핵심 정보

**목적**: 수영·헬스·친구 모임 주최자가 공지, 참여자 관리, 카풀, 정산을 하나의 웹에서 처리할 수 있는 MVP.

**타겟 사용자**: 10~30명 규모의 일회성 소모임을 운영하는 주최자 및 참여자.

---

                                          ┌───────────────┘

## 페이지별 상세 기능

---

### 랜딩 페이지 (`/`)

**역할**: 비인증/인증 모든 사용자

**진입 조건**: 누구나 접근 가능. 인증된 사용자는 `/events`로 리다이렉트.

**사용자 행동**:

- 로그인 버튼 클릭 → `/auth/login`
- 회원가입 버튼 클릭 → `/auth/sign-up`

**기능 목록**:

- 서비스 소개 문구 노출
- 로그인/회원가입 CTA 버튼
- 인증 상태 감지 후 리다이렉트

**구현 기능 ID**: 없음 (라우팅 전용)

---

### 초대 수락 페이지 (`/invite/[token]`)

**역할**: 비인증/인증 모든 사용자 (비인증 접근 허용)

**진입 조건**: 유효한 초대 토큰 보유. 만료/사용된 토큰은 오류 안내.

**사용자 행동**:

- 이벤트 정보 확인
- 수락 버튼 클릭 → 비인증 시 로그인/회원가입 → 자동 참여 처리
- 거절 버튼 클릭 → 초대 거절 처리 후 안내 메시지

**기능 목록**:

- 초대 토큰 유효성 검증
- 이벤트 기본 정보 표시 (제목, 일시, 장소, 주최자)
- 수락 시 `event_members` 레코드 생성 + `invitations.status` → accepted
- 거절 시 `invitations.status` → declined
- 중복 수락 방지 처리

**구현 기능 ID**: F009

---

### 이벤트 목록 페이지 (`/events`)

**역할**: 인증된 모든 사용자

**진입 조건**: 로그인 필수. 미인증 시 `/auth/login` 리다이렉트.

**사용자 행동**:

- 이벤트 카드 클릭 → 해당 이벤트 개요 페이지 이동
- 이벤트 생성 버튼 클릭 → `/events/new`

**기능 목록**:

- 내가 참여 중인 이벤트 목록 표시 (event_members 기준)
- 이벤트 상태(active / cancelled / completed) 배지 표시
- 이벤트별 날짜, 장소, 참여 인원 수 미리보기
- 이벤트 없을 시 빈 상태 안내 + 생성 유도

**구현 기능 ID**: F005

---

### 이벤트 생성 페이지 (`/events/new`)

**역할**: 인증된 모든 사용자 (생성 시 자동으로 host 역할 부여)

**진입 조건**: 로그인 필수.

**사용자 행동**:

- 폼 입력 후 생성 → 생성된 이벤트 개요 페이지로 이동
- 취소 → 이전 페이지(`/events`)로 이동

**기능 목록**:

- 필수 입력: 제목, 시작 일시
- 선택 입력: 설명, 장소, 종료 일시, RSVP 마감일
- 정산 활성화 스위치 (has_expense)
- 카풀 활성화 스위치 (has_carpool)
- zod 유효성 검사 (제목 필수, 종료 일시 > 시작 일시)
- 생성 성공 시 toast 알림

**구현 기능 ID**: F001

---

### 이벤트 개요 페이지 (`/events/[eventId]`)

**역할**: 이벤트 멤버 전체 / host·co_host (수정·상태 변경 권한)

**진입 조건**: `event_members`에 등록된 사용자만 접근 가능.

**사용자 행동 (host/co_host)**:

- 수정 버튼 → 이벤트 정보 수정 폼 표시 (인라인 또는 다이얼로그)
- 취소 버튼 → 이벤트 상태를 cancelled로 변경
- 완료 버튼 → 이벤트 상태를 completed로 변경

**사용자 행동 (member)**:

- RSVP 버튼 클릭 → 참석/불참/미정 선택

**기능 목록**:

- 이벤트 제목, 설명, 일시, 장소, 상태 표시
- 참석 인원 요약 (attending N / total M)
- 나의 RSVP 상태 표시 및 변경
- host/co_host: 이벤트 수정 (제목, 설명, 일시, 장소, RSVP 마감일)
- host: 이벤트 취소/완료 상태 변경 (확인 다이얼로그 포함)
- 취소/완료 상태에서는 수정 불가

**구현 기능 ID**: F002, F003, F004, F006, F010

---

### 참여자 목록 페이지 (`/events/[eventId]/members`)

**역할**: 이벤트 멤버 전체 / host·co_host (초대·역할 변경 권한)

**진입 조건**: `event_members`에 등록된 사용자만 접근 가능.

**사용자 행동 (host/co_host)**:

- 초대 버튼 → 이메일 입력 다이얼로그 → 초대 메일 발송
- 멤버 역할 변경 (host → co_host → member 순환, host만 co_host 임명 가능)

**사용자 행동 (member)**:

- 멤버 목록 조회
- 자신의 RSVP 상태 변경

**기능 목록**:

- 멤버 목록: 이름, 역할 배지, RSVP 상태, 아바타 표시
- 역할별 필터 (host/co_host/member)
- 초대 다이얼로그: 이메일 입력, 중복/기존 멤버 여부 검증, 발송
- 대기 중인 초대 목록 표시 (pending invitations)
- host/co_host: 멤버 역할 변경 (드롭다운 메뉴)
- RSVP 상태 인라인 변경

**구현 기능 ID**: F007, F008, F010

---

### 공지 목록 페이지 (`/events/[eventId]/announcements`)

**역할**: 이벤트 멤버 전체 / host·co_host (작성 권한)

**진입 조건**: `event_members`에 등록된 사용자만 접근 가능.

**사용자 행동 (host/co_host)**:

- 공지 작성 버튼 → 제목·본문 입력 폼 표시 (다이얼로그 또는 인라인)
- 작성 후 저장

**사용자 행동 (member)**:

- 공지 목록 조회
- 공지 클릭 → 본문 확인 (다이얼로그 또는 펼침)

**기능 목록**:

- 공지 목록: 제목, 작성자, 작성일 표시 (최신순)
- 공지 본문 조회 (다이얼로그 또는 accordion)
- host/co_host: 공지 작성 폼 (제목 필수, 본문 필수, textarea)
- 공지 없을 시 빈 상태 안내

**구현 기능 ID**: F011, F012

---

### 정산 현황 페이지 (`/events/[eventId]/expense`)

**역할**: 이벤트 멤버 전체 / host·co_host (비용 입력 권한)

**진입 조건**: `event.has_expense = true`인 이벤트의 멤버만 접근 가능.

**사용자 행동 (host/co_host)**:

- 비용 추가 버튼 → 결제자·항목명·금액 입력 다이얼로그
- 개별 항목 정산 완료 처리 (checkbox)

**사용자 행동 (member)**:

- 정산 목록 조회
- 내가 내야 할 금액 확인
- 최소 이체 결과 확인

**기능 목록**:

- 지출 목록: 항목명, 결제자, 금액, 정산 완료 여부
- 총 지출 합계 표시
- 참석 인원 기준 1인당 금액 계산 (n등분)
- 최소 이체 정산 결과: 누가 누구에게 얼마를 보내야 하는지 목록
- host/co_host: 비용 입력 다이얼로그 (결제자 선택, 항목명, 금액 입력)
- host/co_host: 개별 지출 항목 정산 완료 체크

**구현 기능 ID**: F013, F014, F015, F016

---

### 카풀 목록 페이지 (`/events/[eventId]/carpool`)

**역할**: 이벤트 멤버 전체 / 드라이버 (제안 등록 권한) / 드라이버 (신청 확인 권한)

**진입 조건**: `event.has_carpool = true`인 이벤트의 멤버만 접근 가능.

**사용자 행동 (모든 멤버)**:

- 카풀 제안 등록 버튼 → 출발지·좌석수·메모 입력 다이얼로그
- 다른 멤버의 카풀 제안에 탑승 신청

**사용자 행동 (드라이버)**:

- 내 카풀에 신청한 멤버 목록 조회
- 신청 수락/거절

**기능 목록**:

- 카풀 제안 목록: 드라이버 이름, 출발지, 남은 좌석, 메모 표시
- 탑승 신청 버튼 (이미 신청한 경우 상태 표시)
- 카풀 제안 등록 다이얼로그 (출발지 필수, 좌석수 1-8 선택, 메모 선택)
- 드라이버: 내 카풀 신청자 목록 + 수락/거절 버튼
- 좌석 초과 신청 방지
- 본인 카풀에 본인 탑승 신청 방지

**구현 기능 ID**: F017, F018, F019

## shadcn/ui 컴포넌트

### 신규 설치 필요

없음. 기획된 모든 기능은 기존 설치 컴포넌트로 구현 가능하다.

### 기존 컴포넌트 활용

| 컴포넌트                 | 사용 위치                                          |
| ------------------------ | -------------------------------------------------- |
| `button`                 | 전 페이지 CTA, 수락/거절, 신청 버튼                |
| `card`                   | 이벤트 목록 카드, 카풀 제안 카드                   |
| `dialog`                 | 이벤트 수정, 초대, 비용 입력, 카풀 등록, 공지 작성 |
| `input`                  | 이벤트 생성/수정 폼, 초대 이메일 입력              |
| `label`                  | 폼 필드 레이블                                     |
| `textarea`               | 이벤트 설명, 공지 본문, 카풀 메모                  |
| `select`                 | 역할 변경, 결제자 선택, 좌석 수 선택               |
| `tabs`                   | 이벤트 상세 탭 (개요/참여자/공지/정산/카풀)        |
| `table`                  | 참여자 목록, 정산 목록, 최소 이체 결과             |
| `avatar`                 | 멤버 프로필 이미지                                 |
| `badge` (separator 활용) | 이벤트 상태, 역할, RSVP 상태 표시                  |
| `checkbox`               | 정산 완료 처리                                     |
| `switch`                 | 이벤트 생성 시 정산/카풀 옵션                      |
| `dropdown-menu`          | 멤버 역할 변경, 이벤트 더보기 메뉴                 |
| `skeleton`               | 데이터 로딩 중 플레이스홀더                        |
| `sonner`                 | 성공/오류 toast 알림                               |
| `separator`              | 섹션 구분선                                        |
